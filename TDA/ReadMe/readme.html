General Information

In the following we describe this software as TDA.
XML files are the files provided by medatixx GmbH & Co. KG and each file will be parsed and displayed as one TestRun. One TestRun consists of several UnitTests testing different classes.
TDA shall help visualize all this information and show possible correlations between different TestRuns per class.

User Manual

2.1 Load XML Files

To Load XML Files into the TDA there are two ways:

  1. On startup click the folder button to select a folder and parse the folder and all underlying folders in recursive strategy or click the file button to select one or more specific XML files to be parsed.
  2. On the Menubar click File -> Open folder or Open file

If successful, you will then see the sidebar being updated with information.

2.2 Display one TestRun in the class table

To display a testrun in the table you have to click on a testrun shown in the sidebar with the TestRun - Tab active. It will automatically sort all tested classes by this testrun by its failure percentage(fp) and highlight classes red with a fp higher than 75% and yellow with a fp higher than 50%.

2.3 Display the change of a class' failure percentage over all testruns (chart)

There are two ways to add a class to the chart:

  1. Open the "Classes" tab in the left sidebar. Then select a class by a single click on its name in the package view.
  2. Open the "Table" tab in the main window. Then select a class by a double click on its name in the table.

Both actions will show the desired chart in the "Chart" tab on the main window. You can add several classes to the chart for comparison by following the same procedure.
The chart may be resetted by using the button "Clear Data" underneath the chart.

2.4 Compare a class in two different testruns

To compare different testruns right-click on the corresponding node on the line chart and add it to one of the comparison slots. When both are filled you can use the "Compare" button to see directly which UnitTests failed or passed now and before.
Note: You can not compare two different classes.

2.5 Dependency analysis of failed classes

To see the dependencies between two failed classes click on the Analyzer Tab on the main window. There you will see the Strong Rule. In each row there are classes that are probably dependend with a given confidence. Use the sliders to set the minimum confidence aswell as the minimum distance of two classes.
